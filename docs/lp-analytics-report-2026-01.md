## 5. 問題の根本原因

### 5.1 なぜインデックスされないのか

Googleが「クロール済み - インデックス未登録」と判定する主な理由:

1. **コンテンツの価値が低い**
   - 検索ユーザーが求める情報がない
   - 他サイトと差別化されていない
   - 薄いコンテンツ（thin content）

2. **内部リンクが弱い**
   - ページ間の関連性が低い
   - リンクジュースが分散

3. **外部リンクがない**
   - 他サイトからの被リンクがない
   - ドメインオーソリティが低い

### 5.2 なぜトラフィックがミスマッチなのか

| 仮説 | 可能性 |
|------|--------|
| ボットトラフィック | 高 |
| VPN経由の日本ユーザー | 低 |
| 海外在住日本人 | 中 |
| 偶発的アクセス | 中 |

**推奨**: GA4でボットフィルタを有効化して検証

## 6. 改善提案

### 6.1 緊急対応（優先度: 高）

#### 提案1: コンテンツ強化
- **内容**: トップページに検索されるコンテンツを追加
- **追加すべきコンテンツ**:
  - 「表彰状の作り方」ステップバイステップガイド
  - 「表彰状の書き方・例文集」
  - 「テンプレート別の活用シーン」
- **期待効果**: インデックス登録、検索流入増加

#### 提案2: ブログ/記事ページの追加
- **内容**: SEO向けの記事コンテンツを追加
- **記事案**:
  - 「卒業式・入学式で使える表彰状テンプレート5選」
  - 「子供を褒める表彰状の書き方とコツ」
  - 「会社の表彰式で使える賞状の作り方」
  - 「手作り表彰状で子供のやる気をアップさせる方法」
- **期待効果**: ロングテールキーワードでの流入獲得

#### 提案3: リダイレクト修正 ✅ 対応済み（2026/01/27）
- **内容**: /contact.html → /contact を301リダイレクト
- **実装**: vercel.jsonに既に設定済みを確認
```json
{
  "redirects": [
    { "source": "/contact.html", "destination": "/contact", "permanent": true }
  ]
}
```

#### 提案4: 再インデックスリクエスト ✅ 対応済み（2026/01/27）
- **内容**: GSCから各ページの「インデックス登録をリクエスト」
- **対象URL**:
  - https://awardcert.honeymarron.com/en/
  - https://awardcert.honeymarron.com/contact
  - https://awardcert.honeymarron.com/privacy-policy
- **実施日**: 2026年1月27日 手動対応

### 6.2 中期対応（優先度: 中）

#### 提案5: ボットフィルタ設定 ✅ 対応済み（2026/01/27）
- **内容**: GA4でボットトラフィックを除外
- **設定場所**: GA4 管理 > データの収集と修正 > データフィルタ
- **実施内容**: 「Internal Traffic」フィルタを「テスト」→「有効」に変更
- **注意**: このフィルタは `traffic_type = internal` のトラフィックを除外。完全なボット対策ではなく、GA4の自動ボットフィルタが主な対策。

#### 提案6: キーイベント設定 ✅ 対応済み（2026/01/27）
- **内容**: App Storeボタンクリックをコンバージョンとして計測
- **実施内容**: GA4 > イベント で「click」イベントをキーイベントとしてマーク
- **備考**: すべてのclickイベントがキーイベントとして計測される（App Storeボタン以外も含む）

#### 提案7: 内部リンク強化 ✅ 対応済み（2026/01/27）
- **内容**: フッターで関連ページへのリンクを増やす
- **実施内容**:
  - サイトマップセクション追加（#features, #screenshots, #faq, #download）
  - サポートセクション追加（プライバシーポリシー、利用規約、ライセンス、お問い合わせ）
  - 言語切替リンク追加（日本語/English）
  - App Storeダウンロードボタン追加（gtag追跡付き）
- **対象ファイル**: index.html, en/index.html, styles.css

### 6.3 長期対応（優先度: 低）

#### 提案8: 被リンク獲得
- **内容**: 関連サイトからの被リンクを獲得
- **方法**:
  - アプリレビューサイトへの掲載依頼
  - プレスリリース配信
  - SNSでの拡散

## 7. 戦略的判断

### 7.1 LP継続 vs 廃止の検討

| オプション | メリット | デメリット |
|-----------|---------|-----------|
| **LP継続・改善** | SEO改善で長期的な流入獲得 | 3-6ヶ月の投資が必要 |
| **LP最小化** | 維持コスト削減 | 検索流入の機会損失 |
| **LP廃止** | リソース節約 | アプリ信頼性に影響の可能性 |

### 7.2 推奨

**短期**: LP継続、最小限の改善（リダイレクト修正、再インデックスリクエスト）
**中期**: 効果を見て判断。3ヶ月後にインデックス状況と流入を再評価
**判断基準**: 3ヶ月後にインデックス登録ページが5ページ以上、検索クリックが月10回以上なければ、LP最小化を検討
